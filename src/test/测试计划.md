# 测试计划文档

## 1. 项目背景
本项目为“智能穿搭推荐系统”，包含网页端与微信小程序两个版本，核心功能包括：
- 用户登录注册
- 虚拟试衣（Virtual Try On）
- 衣橱管理
- LLM 推荐
- AI 模型支持（图像编辑、视觉嵌入、GPT-oSS）

项目周期：**2024年1月19日 – 2月22日**

## 2. 测试目标

| 目标 | 描述 |
|------|------|
| 功能完整性 | 验证所有模块功能按需求实现，无重大缺陷 |
| 系统稳定性 | 前后端接口稳定运行，数据库读写正常 |
| AI 模型可用性 | 模型部署成功，推理响应合理 |
| 用户体验 | 页面交互流畅，操作逻辑清晰 |
| 兼容性 | 支持主流浏览器及微信小程序环境 |

## 3. 测试范围

### ✅ 包含内容：

| 模块 | 测试项 |
|------|--------|
| **前端 & API 文档** | 登录注册流程、首页展示、虚拟试衣界面、衣橱管理、LLM 推荐页 |
| **后端 & 数据库** | FastAPI 接口验证、数据库连接与数据一致性、对象存储上传/下载、向量数据库检索 |
| **AI Models** | Qwen-image-edit-2509（虚拟试衣）、Marqo-fashionSigLIP（视觉嵌入）、GPT-oSS（文本生成） |
| **联调 & 集成测试** | 前后端联动、AI 模型接入、模拟用户行为流 |
| **微信小程序** | 第二阶段开发前准备，预留测试入口 |

### ❌ 不包含内容：
- 性能压测（如并发用户数 > 100）
- 安全渗透测试（未涉及敏感信息处理）
- 移动端适配细节（非重点）

## 4. 测试策略

| 类型 | 方法 | 工具/方式 |
|------|------|-----------|
| **功能测试** | 黑盒测试 + 手动执行用例 | Apifox / 浏览器 |
| **接口测试** | API 自动化脚本（可选） | Postman Collection / Python requests |
| **UI 测试** | 手动点击验证 | Chrome DevTools / 手机真机 |
| **模型测试** | 输入输出对比 + 可视化验证 | 图像比对工具 / 人工评审 |
| **集成测试** | 端到端流程验证 | 模拟用户从登录 → 推荐 → 试衣全流程 |
| **回归测试** | 关键路径重复验证 | 每次代码合并后手动检查核心功能 |
